## 一、功能背景

### 1.1 业务痛点

当前小型工程施工期间的安全监管存在以下问题:

1. **巡查记录不规范**:纸质记录易丢失,信息不完整,无法追溯
2. **隐患整改难跟踪**:发现隐患后缺少闭环管理,整改情况难以跟踪
3. **违规处置不及时**:发现违规行为后处置流程不清晰,容易推诿扯皮
4. **数据统计困难**:巡查数据分散,无法统计分析,难以支撑决策

### 1.2 解决方案

通过"安全巡检"功能,实现:

1. **规范巡查流程**:标准化巡查记录,现场拍照取证,数据实时上传
2. **闭环整改管理**:隐患登记→整改通知→整改提交→验收销号,全流程跟踪
3. **分级处置机制**:根据违规严重程度,自动触发不同处置流程
4. **数据可视化**:巡查数据实时统计,支持导出和分析

## 二、核心概念

### 2.1 工程状态定义

工程共有四种状态，状态流转关系如下：

```
正常施工 ──→ 现场整改 ──→ 正常施工
         └─→ 停工整改 ──→ 正常施工
         └─→ 已核销（终态）
```

---

#### 正常施工

**定义**：工程正常进行，无未处理的安全隐患。

**进入条件**：
- 工程初始状态（开工时默认为正常施工）
- 现场整改或停工整改的整改记录审核通过后，恢复为正常施工

**该状态下可执行的操作**：

| 角色 | 可执行操作 |
|------|-----------|
| 巡查人员 | 发起巡查、登记违规 |
| 监管人员 | 提交完工记录（触发核销） |
| 施工方 | 申请核销 |

**状态流转**：
- → **现场整改**：巡查登记违规，处置方式选择"现场整改"
- → **停工整改**：巡查登记违规，处置方式选择"停工整改"
- → **已核销**：监管提交完工记录，或施工方申请核销且监管审核通过

---

#### 现场整改

**定义**：巡查发现需要限时整改的违规，工程暂停施工，等待整改完成并通过审核。

**进入条件**：巡查登记违规时，"是否需要限时整改"选择"是"，处置方式选择"现场整改"。

**该状态下可执行的操作**：

| 角色 | 可执行操作 |
|------|-----------|
| 巡查人员 | 可继续对该工程发起新的巡查 |
| 施工方 | 提交整改记录（上传整改照片和整改说明） |
| 审核员 | 审核整改记录（通过/驳回） |
| 监管人员 | 查看整改进度、催办整改 |

**状态流转**：
- → **正常施工**：整改记录审核通过
- → **停工整改**：巡查发现更严重违规，处置方式升级为"停工整改"（新违规记录覆盖工程状态）

---

#### 停工整改

**定义**：巡查发现严重违规行为，工程强制停工，必须完成整改并通过审核后方可复工。

**进入条件**：巡查登记违规时，"是否需要限时整改"选择"是"，处置方式选择"停工整改"。

**该状态下可执行的操作**：

| 角色 | 可执行操作 |
|------|-----------|
| 巡查人员 | 可继续对该工程发起新的巡查 |
| 施工方 | 提交整改记录（上传整改照片和整改说明） |
| 审核员 | 审核整改记录（通过/驳回） |
| 监管人员 | 查看整改进度、催办整改、移交执法 |

**状态流转**：
- → **正常施工**：整改记录审核通过

---

#### 已核销

**定义**：工程已完工，核销审核通过后进入该状态。已核销为终态，不可撤销，不可流转至其他状态。

**进入条件**（二选一）：
1. **监管主动核销**：监管人员提交完工记录，直接核销
2. **施工方申请核销**：施工方发起核销申请，监管人员审核通过后核销

**该状态下可执行的操作**：

| 角色 | 可执行操作 | 说明 |
|------|-----------|------|
| 所有角色 | 查看历史巡查记录 | 只读，不可修改 |
| 施工方 | 继续提交未完成的整改记录 | 用于恢复因违规被扣的积分 |
| 巡查人员 | **不可**发起新的巡查 | — |
| 监管人员 | **不可**撤销核销状态 | — |

> **注意**：已核销工程仍可继续走完未结束的整改流程（提交整改记录→审核通过→积分恢复），但不产生新的巡查记录。

### 2.2 违规行为分类

根据是否需要限时整改,违规行为分为两类:

**不需要限时整改的违规**:
- 轻微违规,不影响施工安全
- 勾选后提交审核通过即可加分
- 不提交审核不加分
- 工程状态不改变

**需要限时整改的违规**:
- 存在安全隐患,需要整改后才能继续施工
- 必须提交整改记录并通过审核
- 工程状态变更为"现场整改"或"停工整改"
- 整改通过后恢复"正常施工"状态

### 2.3 角色与权限

| 角色 | 说明 | 核心操作 |
|------|------|----------|
| 巡查人员 | 街道/社区/物业的巡查工作人员 | 发起巡查、登记违规 |
| 审核员 | 与提交整改记录的施工方**同组织**的成员（街道/社区/物业各自内部审核） | 审核整改记录（通过/驳回） |
| 监管人员 | 上级监管部门工作人员 | 查看整改进度、催办整改、移交执法 |
| 施工方 | 施工单位负责人或相关人员 | 查看违规记录、提交整改记录、申请核销 |

**审核规则**：整改审核由**同组织内部**相互审核，即：
- 街道巡查员发起的违规 → 街道审核员审核整改
- 社区巡查员发起的违规 → 社区审核员审核整改
- 物业巡查员发起的违规 → 物业审核员审核整改

### 2.4 巡查频次规则

系统根据工程风险等级和类型，计算每个工程的"应巡查频次"，超出频次要求的工程在**待巡查列表**中提醒，不强制阻断操作。

**频次规则**（参考深圳标准）：

**高风险工程**：

| 组织 | 频次要求 |
|------|----------|
| 街道 | 每10天（含）内至少1条巡查记录 |
| 社区 | 每14天（含）内至少1条巡查记录 |
| 物业 | 每2天（含）内至少1条巡查记录 |

**非高风险工程 · 家装工程**（工程类型为"限额以下竣工后验收合格的室内住宅装修类型"）：

| 组织 | 频次要求 |
|------|----------|
| 街道 | 每30天（含）内至少1条巡查记录 |
| 社区 | 每21天（含）内至少1条巡查记录 |
| 物业 | 每7天（含）内至少1条巡查记录 |

**非高风险工程 · 非家装工程**：

| 组织 | 频次要求 |
|------|----------|
| 街道 | 每21天（含）内至少1条巡查记录 |
| 社区 | 每14天（含）内至少1条巡查记录 |
| 物业 | 每3天（含）内至少1条巡查记录 |

**系统行为**：
- 超出频次要求的工程显示在"待巡查"列表，并标注超期天数
- 仅作提醒，不阻断其他操作
- 待巡查数量在首页概览中统计展示

### 2.5 隐患分级与处置方式映射规则

巡查使用标准化《隐患清单》进行现场核查，每个检查项已预设隐患等级，等级直接映射处置方式：

| 隐患等级 | 定义 | 处置方式 | 对工程状态的影响 |
|---------|------|---------|----------------|
| **较大隐患** | 存在重大安全风险，需立即停工 | 停工整改（强制，不可修改） | 工程状态变为"停工整改" |
| **一般隐患** | 存在一般安全问题 | 由巡查员判断：现场整改 / 不需要限时整改 | 变为"现场整改"，或保持不变 |

**业务规则**：
- 巡查员选择**较大隐患**检查项后，处置方式自动锁定为"停工整改"，不可更改
- 巡查员选择**一般隐患**检查项后，可自行选择"现场整改"或"不需要限时整改"
- 同一次巡查登记多条隐患时，工程状态取最严级别（有一条较大隐患即停工整改）

> 完整隐患清单详见文末附录。

## 三、整体页面结构

```
首页（巡查概览）
  ├─ 当前身份栏（姓名 + 当前组织角色 + "切换身份"入口）
  ├─ 在建工程统计：小型工程 / 住宅装修 / 零星作业 各类型在建总数
  ├─ 待巡查数量（超出频次要求的工程数）
  ├─ 快捷功能区：待巡查列表 / 全部工程 / 整改跟踪
  ├─ 工程状态汇总：正常施工 / 现场整改 / 停工整改
  ├─ 我的巡查统计（当前用户）
  │     ├─ 本月：已完成巡查 / 发现违规 / 待跟进整改
  │     └─ 累计：巡查次数 / 违规条数
  └─ 最近巡查记录（近3条，含工程名、日期、结果、工程状态）

功能区
  ├─ 待巡查列表 → 工程列表（需尽快巡查）
  └─ 全部工程列表
        ├─ 区域选择栏（行政区域切换）
        ├─ 搜索栏（类型下拉 + 输入框 + 搜索按钮）
        ├─ 类型Tab（胶囊样式，含数量）：小型工程 / 住宅装修 / 零星作业
        ├─ 状态Tab（含数量徽章）：正常施工 / 现场整改 / 停工整改 / 已核销
        └─ 工程卡片（含风险标签 + 特殊施工标签 + 施工时间 + 组织统计）
              → 工程详情页
                    ├─ 简化版备案信息（工程名、地址、施工单位、施工时间、负责人）
                    │     └─ "查看完整备案" 按钮 → 完整备案详情页（只读）
                    ├─ 地图（显示工程位置，方便巡查员导航）
                    ├─ 巡查记录列表（多条，倒序）
                    │     └─ 巡查详情页
                    ├─ 违规整改记录列表（按违规条目展示，多条）
                    │     └─ 整改详情页（含逐条整改进度）
                    └─ 操作按钮（根据角色和工程状态显示）
                          ├─ 发起巡查（巡查人员）
                          ├─ 提交整改记录（施工方，有待整改违规时）
                          ├─ 审核整改（审核员，有待审核整改时）
                          ├─ 申请核销（施工方，正常施工状态）
                          └─ 提交完工核销（监管人员，正常施工状态）
```

## 四、功能详细设计

### 4.1 首页概览

#### 4.1.1 页面结构

**页面名称**："安全巡检"首页

**导航栏**：标题"安全巡检"（左），消息通知铃铛图标（右）

**页面模块（从上到下）**：

**① 当前身份栏**（导航栏下方，蓝色渐变底色）

| 元素 | 说明 |
|------|------|
| 头像 | 圆形，取姓名首字，白色字 |
| 姓名 | 如"张三" |
| 当前身份 | 格式："组织名 · 角色"，如"夏阳街道 · 巡查员" |
| 切换按钮 | "切换身份 ▾"，圆角胶囊样式，点击弹出身份选择弹层 |

切换规则：同一账号可绑定多个组织角色；切换后，首页所有数据模块均刷新为所选身份对应组织的数据视角。

---

**② 在建工程统计**（白色卡片）

- 标题行显示总数：如"在建工程统计（共 542 项）"
- 内容区横向三列：
  - 小型工程：X 个（蓝色数字）
  - 住宅装修：X 个（绿色数字）
  - 零星作业：X 个（橙色数字）

---

**③ 待巡查提醒大卡**（橙红渐变卡片）

- 左侧：大号数字 + "个工程待巡查（超出频次要求）"+ ⓘ 说明图标
- 右侧：大号剪贴板图标（装饰）
- 点击卡片主体 → 进入待巡查工程列表
- 点击 ⓘ 图标 → 弹出「各组织巡查频次要求」说明弹层（见 4.1.3）

---

**④ 快捷功能区**（横向三个入口，白色卡片样式）

| 入口 | 图标 | 红点规则 |
|------|------|---------|
| 全部工程 | 🏗 | 无红点 |
| 整改跟踪 | 📝 | 有待处理整改时显示数量红点 |
| 完工核销 | ✅ | 有待审核核销申请时显示数量红点 |

---

**⑤ 工程状态汇总**（白色卡片）

横向三列，统计当前身份所属组织辖区内的工程数量：
- 正常施工：X 个（绿色）
- 现场整改：X 个（橙色）
- 停工整改：X 个（红色）
- 点击各列 → 进入对应状态筛选下的工程列表

---

**⑥ 我的巡查统计**（白色卡片）

标题右侧显示当前月份，如"本月（3月）"。

上半区横向三列（本月数据）：
- 已完成巡查：X 次（蓝色）
- 发现违规：X 条（红色）
- 待跟进整改：X 条（橙色，指自己登记的违规中尚未完成整改的条数）

下半区（灰色细字，累计数据）：
- 累计巡查 X 次 / 累计发现违规 X 条

---

**⑦ 最近巡查记录**（白色卡片）

- 标题右侧有"全部 ›"链接入口
- 展示当前用户最近 3 条巡查记录，每条显示：
  - 彩色圆点（颜色对应巡查结果：绿色=无违规，橙色=现场整改，红色=停工整改）
  - 工程名称
  - 巡查日期 · 巡查结果描述（如"发现违规 2 条"）· 工程状态（带颜色标注）
  - 右侧"›"箭头

---

#### 4.1.2 交互说明

- 点击"切换身份"按钮 → 弹出底部弹层，列出该账号绑定的全部组织角色；当前激活角色标注 ✓；点击其他角色后身份切换，页面数据全部刷新，弹层自动关闭

- 点击工程状态汇总各列 → 进入对应状态筛选下的工程列表
- 点击"整改跟踪"入口 → 进入整改跟踪页面
- 点击"完工核销"入口 → 进入核销管理页面
- 点击最近巡查记录条目 → 进入对应巡查详情页
- 点击最近巡查记录"全部 ›" → 进入个人全部巡查记录列表

> **交叉巡查员说明**：同一账号可绑定多个街道/社区/组织的巡查员或监管人员身份。切换身份后，所有数据（工程统计、待巡查提醒、状态汇总、个人巡查统计）均切换为所选身份所在组织的数据视角。

| 工程分类 | 颜色标记 | 街道频次 | 社区频次 | 物业频次 |
|---------|---------|---------|---------|---------|
| 高风险工程 | 🔴 红色 | 每 10 天至少巡查 1 次 | 每 14 天至少巡查 1 次 | 每 2 天至少巡查 1 次 |
| 非高风险 · 住宅装修工程 | 🟡 黄色 | 每 30 天至少巡查 1 次 | 每 21 天至少巡查 1 次 | 每 7 天至少巡查 1 次 |
| 非高风险 · 其他工程 | 🟢 绿色 | 每 21 天至少巡查 1 次 | 每 14 天至少巡查 1 次 | 每 3 天至少巡查 1 次 |

底部说明：超出频次要求的工程将在「待巡查」列表中提醒，不影响其他操作。

### 4.2 工程列表页

#### 4.2.1 页面结构

**页面名称**："工程列表"页面

**页面功能**：

1. **区域选择栏**
   - 显示当前所选行政区域（如"南山区"），可点击切换
   - 位于页面顶部，方便跨区域巡查人员快速切换辖区

2. **搜索栏**
   - 搜索类型下拉（工程名称 / 工程地址，默认"工程名称"）
   - 文本输入框
   - "搜索"按钮

3. **工程类型 Tab**（胶囊样式，3个，Tab 标题附带数量）
   - 小型工程(X) / 住宅装修(X) / 零星作业(X)

4. **工程状态 Tab**（下划线样式，4个，Tab 标题附带数量徽章）
   - 正常施工 · X / 现场整改 · X / 停工整改 · X / 已核销 · X

5. **工程卡片列表**

   **基本信息区**：
   - 工程名称

### 4.5 违规登记

#### 4.5.1 页面结构

**页面名称**："违规登记"页面

**设计原则**：**每条违规独立登记**，一次巡查可添加多条违规记录，每条违规独立设置整改要求、期限和处置方式。一次巡查产生的所有需限时整改的违规，施工方合并提交**一条整改记录**（包含每条违规各自的整改照片和说明），审核员对该整改记录**整体通过或整体驳回**；驳回后施工方重新提交整个整改记录。

**页面功能**：

**顶部工程信息条**（只读）
- 工程名称

**违规记录卡片列表**（可添加多条，每条为一个独立卡片）

每条违规卡片包含：

1. **隐患选择**（必填，从标准隐患清单中选择）
   - 按类别展示检查项（信息登记管理 / 安全管理 / 高处作业 / 施工用电 / 施工机具 / 消防安全 / 物体打击 / 文明施工 / 拆除作业 / 脚手架 / 吊篮作业 / 起重吊装作业 / 有限空间作业 / 物料提升机）
   - 选中后自动带出"隐患等级"标签（较大隐患 / 一般隐患）
   - 支持搜索关键字快速定位检查项
   - 可选"其他"并手动输入描述（手动输入默认为一般隐患）

2. **违规详情**
   - 违规描述（文本输入，限500字，选填，用于补充说明）
   - 现场照片（必传，最多9张）

3. **整改要求**
   - **较大隐患**：处置方式自动锁定为"停工整改"，不可更改；整改期限必填
   - **一般隐患**：需选择处置方式（单选）
     - 不需要限时整改 → 此条隐患不影响工程状态，审核通过加分
     - 现场整改 → 工程状态变为"现场整改"，整改期限必填
   - 整改要求说明（文本输入，限500字）

**底部操作**
- "＋ 继续添加违规" 按钮（添加下一条违规卡片）
- "提交违规登记" 按钮

#### 4.5.2 工程状态变更规则

同一次巡查中登记多条违规时，工程状态取**最严级别**：
- 有任意一条选择"停工整改" → 工程状态变为"停工整改"
- 全部为"现场整改" → 工程状态变为"现场整改"
- 全部为"不需要限时整改" → 工程状态不变

#### 4.5.3 交互说明

- 每条违规卡片可独立展开/折叠
- 提交后，本次巡查的所有违规合并为一条整改任务，通知施工方统一处理

### 4.6 整改提交（施工方）

#### 4.6.1 页面结构

**页面名称**："提交整改记录"页面

**触发条件**：施工方收到整改通知后进入，或在工程详情页点击"提交整改记录"按钮

**数据关系**：一条巡查记录对应一条整改记录（1:1）；整改记录内按违规条目分组展示。

**页面功能**：

**顶部巡查信息头**（只读，蓝色背景卡片）

- 巡查时间 / 工程名称

**违规整改列表**（每条违规一个独立分区，不可删减）

每条违规分区包含两部分：

① **违规信息**（只读）
- 违规类型标签（颜色标注处置方式：停工整改/现场整改）
- 违规描述
- 整改要求说明
- 整改期限（标注剩余天数，逾期时标红）

#### 4.6.2 交互说明

#### 4.7.1 页面结构

**页面名称**："审核整改记录"页面

**违规整改内容列表**（每条违规一个分区，顺序与巡查记录一致）

每条违规分区包含两部分：

① **原始违规信息**（只读）
- 违规类型标签
- 违规描述
- 整改要求 / 整改期限

#### 4.7.2 交互说明

- 审核通过 → 整改记录标记为已完成，施工方积分恢复，工程状态恢复"正常施工"，通知施工方
- 审核驳回 → 整改记录退回"待整改"状态，施工方收到驳回通知（含驳回原因），可修改后重新提交整条记录

## 五、业务流程

### 5.1 完整流程图

```
发起巡查（从工程详情页进入，工程已确定）
  │
  ├─ 未发现违法违规 → 提交巡查记录 → 不改变工程状态 → 结束
  │
  └─ 发现违法违规行为 → 逐条登记违规（每条独立设置整改要求）
         │
         ├─ 含"不需要限时整改"的违规（可单独存在或与限时整改同时存在）
         │    └─ 记录在案，不影响工程状态，不触发整改流程
         │
         ├─ 含"现场整改"违规（无停工整改违规时）
         │    └─ 工程状态变更为"现场整改"
         │
         └─ 含"停工整改"违规（优先级最高）
               └─ 工程状态变更为"停工整改"

【整改流程（针对本条巡查记录中所有限时整改违规）】

巡查记录提交后 → 通知施工方
  │
  └─ 施工方提交整改记录（一条，包含每条限时整改违规各自的照片和说明）
         │
         └─ 同组织审核员审核（整体操作）
               │
               ├─ 整体通过（上传验收照片）
               │    └─ 工程状态恢复"正常施工" → 施工方积分恢复 → 结束
               │
               └─ 整体驳回（填写驳回原因）
                    └─ 通知施工方 → 施工方修改后重新提交整条记录（循环）
```

### 5.2 整改逾期处理流程

```
整改期限到期（以整条整改记录的整改期限为准，取所有违规中最早的期限）
  │
  ├─ 施工方已提交整改记录 → 等待同组织审核员审核（不算逾期）
  │
  └─ 施工方未提交整改记录
       │
       ├─ 逾期1-3天 → 系统自动发送催办通知给施工方
       │
       ├─ 逾期3-7天 → 监管人员/巡查员可手动催办
       │
       └─ 逾期超过7天 → 可移交执法部门
```

### 5.3 工程核销流程图

```
工程正常施工状态（无未处理整改）
  │
  ├─ 路径一：监管主动核销
  │    └─ 监管人员提交完工记录 → 工程状态变更为"已核销"→ 结束
  │
  └─ 路径二：施工方申请核销
       └─ 施工方发起核销申请
            └─ 监管人员审核
                 ├─ 审核通过 → 工程状态变更为"已核销"→ 结束
                 └─ 审核驳回 → 通知施工方 → 结束

注：
- 已核销为终态，不可撤销
- 已核销工程存在未完成整改记录时，仍可继续提交整改→审核通过→积分恢复
- 已核销工程不可发起新的巡查
```

## 七、技术实现要点

### 7.1 工程状态管理

- 工程状态字段：status（正常施工/现场整改/停工整改/已核销）
- 状态变更规则：取同一次巡查中所有违规的最严处置方式
- 状态变更记录：记录状态变更时间、变更原因、操作人
- 状态联动：工程状态变更后，自动通知相关方
- 已核销为终态：数据库层面限制已核销工程的状态字段不可被修改

### 7.2 违规整改逐条管理

- 每条违规作为独立记录（violation），含：违规类型、描述、照片、整改要求、期限、处置方式
- 每条违规有独立整改记录（rectification），含：施工方提交内容、审核结果、审核照片
- 工程状态由所有**未完成的限时整改违规**中最严级别决定
- 所有限时整改违规均审核通过 → 工程状态恢复正常施工

### 7.3 整改期限提醒

- 整改期限到期前1天:发送提醒通知给施工方
- 整改期限到期:标记为逾期,发送催办通知
- 逾期超过设定天数:自动触发移交执法流程

### 7.4 照片上传与存储

- 支持拍照和相册选择
- 照片压缩后上传,减少流量消耗
- 照片存储在云端,支持查看和下载

### 7.5 消息通知

- 整改通知（逐条）：每条违规登记后分别发送给施工方
- 审核结果通知（逐条）：每条整改审核完成后发送给施工方
- 催办通知：发送给施工方
- 逾期预警：发送给监管人员
- 待巡查提醒：发送给对应组织的巡查人员

### 7.6 审核权限控制

- 审核员只能审核**本组织提交整改**记录的违规
- 同一人不能既提交整改又审核同一条违规
- 权限与备案模块的组织归属字段联动
